# Инструкция по установке и запуску

## После загрузки файлов из репозитория:

### 1. Установите зависимости

```bash
npm install
```

Это установит все необходимые пакеты из `package.json`:
- express (сервер)
- sqlite3 (база данных)
- bcryptjs (хеширование паролей)
- jsonwebtoken (JWT токены)
- cors (для CORS)
- axios (HTTP запросы)
- uuid (генерация ID)
- dotenv (переменные окружения)

### 2. Создайте файл .env

Скопируйте `env.example` в `.env`:

```bash
cp env.example .env
```

Или создайте файл `.env` вручную со следующим содержимым:

```env
PORT=3000
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
YOOKASSA_SHOP_ID=1221819
YOOKASSA_SECRET_KEY=test_2hVHBJNfZcW8GuLJd2X6E037JgUVcH_nYKl1_IR3028
```

### 3. Запустите сервер

**Режим разработки (с автоперезагрузкой):**
```bash
npm run dev
```

**Продакшн режим:**
```bash
npm start
```

### 4. Откройте в браузере

```
http://localhost:3000
```

## Что создается автоматически:

- `database.db` - база данных SQLite (создается при первом запуске)
- Таблицы в БД (users, payments, service_requests)

## Важные файлы:

✅ **Должны быть в репозитории:**
- `package.json` - зависимости
- `server.js` - сервер
- Все HTML файлы
- Все JS файлы (auth.js, payment.js, smooth-scroll.js)
- `style.css`
- `env.example` - пример переменных окружения
- `.gitignore` - исключает ненужные файлы

❌ **НЕ должны быть в репозитории (в .gitignore):**
- `.env` - секретные ключи (создается локально)
- `node_modules/` - зависимости (устанавливаются через npm install)
- `database.db` - база данных (создается автоматически)

## Если что-то не работает:

1. Проверьте, что Node.js установлен: `node --version`
2. Проверьте, что все зависимости установлены: `npm list`
3. Проверьте, что файл `.env` создан и содержит правильные ключи
4. Проверьте логи сервера в консоли

## Для продакшена:

1. Измените `JWT_SECRET` на случайную строку
2. Замените тестовые ключи ЮKassa на реальные
3. Настройте HTTPS
4. Используйте переменные окружения на сервере
